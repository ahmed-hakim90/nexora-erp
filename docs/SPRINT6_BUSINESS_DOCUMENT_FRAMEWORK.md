# Sprint 6: Business Document Framework

Sprint 6 adds the generic document foundation that future ERP modules can reuse. It intentionally does not implement invoices, orders, stock transfers, production, HR requests, vouchers, expenses, or accounting documents.

## Scope

- Generic document registry: `document_types` and `document_statuses`.
- Generic document shell: `business_documents`.
- Generic relations: references, timeline events, comments, file attachment relations, print snapshots, and export job placeholders.
- Document type and reference keys validate generic key shape only, so future modules can register their own document types without Sprint 6 implementing those modules.
- Numbering integration through the existing `numbering_sequences` engine and the registry-backed `generate_business_document_number` RPC.
- Service-layer operations for creating shells, updating metadata, changing status, comments, references, attachment relations, print placeholders, and export placeholders.
- Permission-gated RLS and server-side permission checks for `documents.*` permissions.

## Boundaries

- No document-specific business tables are created.
- Attachments only reference the platform file engine through `file_attachments`.
- Generic references do not add foreign keys to future business tables; only document-to-document references enforce a `business_documents` target.
- Print and export records are metadata/job placeholders only; no PDFs or files are generated by Sprint 6.
- Services do not import future business modules.
- UI and app routes do not query Supabase directly.
